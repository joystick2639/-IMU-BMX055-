# -IMU-BMX055-
1. Цель проекта
Разработать программное обеспечение на Python для считывания данных с датчика BMX055 (акселерометр, гироскоп, магнитометр), их калибровки и слияния с помощью фильтра Маджвика для получения точной ориентации в пространстве (углы Эйлера: крен, тангаж, рыскание). Конечная цель – использование в ROS2 для навигации мобильных роботов. Ключевая задача – обеспечить выдачу относительного угла поворота (yaw) от начального положения, что критично для систем управления.

2. Аппаратная часть
Датчик: BMX055 (Bosch Sensortec), объединяющий акселерометр (BMA280), гироскоп (BMG160) и магнитометр (BMM150).
Интерфейс: I²C, адреса устройств:
Акселерометр: 0x19
Гироскоп: 0x69
Магнитометр: 0x13
Подключение: Датчик подключён к шине I²C микрокомпьютера (например, Raspberry Pi) с подтяжкой линий SDA/SCL к 3.3 В. В коде используется библиотека smbus2 для доступа к шине.

3. Программная реализация
Проект реализован на Python 3 и состоит из следующих модулей (классов) и функций:

3.1 Классы датчиков
Каждый датчик инкапсулирован в отдельный класс с методами инициализации и чтения данных.

Accel – акселерометр:
begin(): проверка ID, сброс, настройка диапазона (±2g) и полосы пропускания (125 Гц).
read(): чтение 7 байт данных, преобразование в 16-битные значения с учётом дополнения до двух, масштабирование в единицы g (масштаб 2g/2048).
get_angles(): вычисление углов крена и тангажа по формулам из акселерометра (используется для начальной инициализации).

Gyro – гироскоп:
begin(): проверка ID, сброс, настройка диапазона ±125°/s и полосы 23 Гц.
read(): чтение 6 байт, преобразование в радианы/сек (масштаб 124.87/32768).
calibrate(): сбор 100 измерений в покое, усреднение для определения нулевых смещений.
read_calibrated(): чтение и вычитание смещений.

Mag – магнитометр:
begin(): включение питания, проверка ID, настройка разрешения, чтение калибровочных коэффициентов из памяти датчика (для температурной компенсации).
read_raw_compensated(): чтение 8 байт данных, выполнение температурной компенсации согласно документации BMM150, возврат значений в миллигауссах (мГс).
Калибровка hard-iron bias: в проекте используются фиксированные смещения, полученные ранее (X=1235.0, Y=1220.0, Z=-2200.0 мГс). Для их определения ранее выполнялась процедура вращения датчика во всех направлениях и вычисление центра эллипса.

3.2 Преобразование данных
Акселерометр: единицы – g (1g ≈ 9.81 м/с²). Для фильтра используются непосредственно эти значения (нормализация выполняется внутри фильтра).

Гироскоп: значения в °/с переводятся в рад/с умножением на math.radians() и, при необходимости, инвертируются знаком GYRO_SIGN.

Магнитометр:
Сырые компенсированные значения в мГс (raw_x, raw_y, raw_z) вычитаются смещения bias.
Далее применяется перестановка осей в зависимости от параметра MAG_OPTION (по умолчанию 1: X = -Y_raw, Y = +X_raw, Z = Z_raw), что соответствует физической ориентации датчика на плате.
Перевод в микротесла (µT) умножением на MG_TO_UT = 0.1 (1 мГс = 0.1 мкТл). Эти значения подаются в фильтр.

3.3 Фильтр Маджвика (собственная реализация)
Класс MadgwickFilter реализует алгоритм слияния данных, предложенный Sebastian Madgwick. Он обновляет кватернион ориентации на основе измерений гироскопа и корректирует его по акселерометру и магнитометру.

Параметры:
beta = 1.0 – коэффициент скорости сходимости по акселерометру/магнитометру. Значение 1.0 обеспечивает быструю коррекцию без внесения значительного шума.
Метод update(ax, ay, az, gx, gy, gz, mx, my, mz, dt):
Нормализация векторов акселерометра и магнитометра.
Вычисление градиента ошибки между измеренными направлениями и направлениями, предсказанными текущим кватернионом.
Расчёт производной кватерниона от угловых скоростей.
Интегрирование производной с вычитанием градиента, умноженного на beta и шаг времени dt.
Нормализация кватерниона.
Метод get_euler() преобразует кватернион в углы Эйлера (roll, pitch, yaw) в градусах.

3.4 Инициализация кватерниона
Для исключения начального дрейфа и обеспечения соответствия реальному положению выполняется инициализация кватерниона по первым измерениям:
По акселерометру вычисляются углы roll0 и pitch0.
По магнитометру (после вычитания bias и перестановки) вычисляется yaw0.
Углы Эйлера преобразуются в кватернион с правильным порядком компонент (w, x, y, z).

Этот кватернион присваивается фильтру.


3.5 Относительный угол рыскания
Поскольку для ROS2 требуется изменение угла относительно старта, введена переменная initial_fyaw. После первого достоверного измерения (когда fyaw не равен нулю) он запоминается. Далее для каждой итерации вычисляется:

python
f_yaw_rel = (initial_fyaw - fyaw) % 360   # по часовой стрелке положительное
Это значение и выводится как Rel Yaw.

3.6 Вывод данных
В бесконечном цикле программа выводит:
Acc(g): акселерометр в g.
Gyr(°/s): гироскоп в °/с.
Raw (mG): сырые компенсированные данные магнитометра (до вычитания bias и перестановки).
Corr (µT): данные после вычитания bias и перестановки (в микротеслах).
Yaw_abs: абсолютный курс, вычисленный по atan2(my, mx).
Yaw_rel: относительный курс от начального направления (0° = старт).
Filtered (deg): углы после фильтра (roll, pitch, yaw), а также относительный yaw фильтра Rel Yaw.
Частота вывода ~20 Гц (sleep 0.05 с).
